## Архитектура системы

### Поток данных

```
WhatsApp → Hermes Bridge (:3000) → bridge_wrapper.py → main_waha.py (poll 3s)
  → Guard → Router → [QA/DB/Weather/Grok/Schedule/Poll] → Reply
                          │
                          ▼
                    QA-парсер (qa.py)
                          │
                    bot_memory_facts
                          │
            ┌─────────────┼─────────────┐
            ▼             ▼             ▼
   ┌──────────────┐ ┌────────────┐ ┌──────────┐
   │ ojr_section1 │ │ojr_section3│ │  ojr_    │
   │ _personnel   │ │_work_log   │ │ weather  │
   └──────────────┘ └─────┬──────┘ └──────────┘
                          │
            ┌─────────────┼─────────────┐
            ▼             ▼             ▼
       photo_log    daily_summary    materials
                          │
                          ▼
                   ЕЖО (fill_ejo.py) = view на ojr_section3_work_log
```

### Компоненты

| Компонент | Назначение | Технология |
|-----------|------------|------------|
| **Hermes Bridge** | WhatsApp-мост (WebSocket) | Node.js, systemd, порт 3000 |
| **bridge_wrapper.py** | Адаптер моста | Python 3.11+ |
| **main_waha.py** | Ядро бота, цикл опроса | Python 3.11+, асинхронный |
| **router.py** | Маршрутизация сообщений | Python |
| **qa.py** | Парсер строительных сообщений | Python, регулярные выражения |
| **fill_ejo.py** | Генератор Excel-отчётов | Python, openpyxl |
| **document_extractor.py** | Извлечение данных из документов | Python, порт 8099 |
| **PostgreSQL** | База данных ОЖР (14 таблиц ГОСТ) | PostgreSQL 15+ |

## WhatsApp-подключение

### Принцип работы

Система подключается к WhatsApp через Hermes Bridge — WebSocket-мост, работающий как systemd-сервис. Это не облачный сервис: сессия WhatsApp хранится локально на сервере.

### Технические требования

- Отдельная SIM-карта (не личный номер сотрудника)
- Номер телефона не должен быть заблокирован в WhatsApp
- Телефонный аппарат для первоначальной активации (возвращается заказчику)

### Варианты подключения

**Вариант 1: QR-код (рекомендованный)**

1. SIM-карта вставляется в телефон с WhatsApp
2. Мы генерируем QR-код на сервере
3. На телефоне: WhatsApp → Связанные устройства → Сканировать QR
4. После привязки SIM можно извлечь — система работает через интернет
5. Телефон возвращается заказчику

**Вариант 2: Прямое подключение**

1. SIM-карта остаётся в телефоне
2. Телефон всегда включён и подключён к интернету
3. Используется как шлюз

### WhatsApp-группы

| Группа | Назначение | Поведение системы |
|--------|------------|-------------------|
| **Песочница** | Тестирование и отладка | Читает и отвечает |
| **Боевая** | Рабочая коммуникация | **Только читает** — не пишет без явной команды руководителя |

## Telegram-подключение

### Принцип работы

Руководитель управляет системой через Telegram. На сервере работает агент Hermes — это «мозг» системы. Команды отправляются с префиксом «Алихан».

### Настройка

1. Создать бота через [@BotFather](https://t.me/BotFather)
2. Получить токен бота
3. Передать токен нам (или добавить в конфигурацию сервера)
4. Бот готов к работе — занимает 2 минуты

### Команды управления

| Команда | Назначение |
|---------|------------|
| «Алихан, запускай опрос» | Начать ежедневный сбор данных |
| «Алихан, статус опроса» | Проверить прогресс опроса |
| «Алихан, закончить опрос» | Финализировать опрос |
| «Алихан, снимок дня» | Сводка за день |
| «Алихан, формируй ЕЖО» | Excel-отчёт за дату |
| «Алихан, формируй АВР» | Акт выполненных работ |
| «Алихан, что делали 5 июля?» | Поиск по архиву |

## Безопасность

### Хранение данных

- Сессия WhatsApp хранится только на сервере, не в облачных сервисах
- Все ключи доступа — в переменных окружения (`systemd` environment), не в коде
- Доступ к PostgreSQL — только с localhost (не exposed наружу)
- `pg_hba.conf` настроен на `127.0.0.1/32` для пользователя `evolution`

### Сетевая безопасность

- Открытые порты: только 22 (SSH), 3000 (WhatsApp Bridge, localhost-only), 8099 (extractor, localhost-only)
- Рекомендуется fail2ban для SSH
- UFW: разрешить только 22/tcp, остальное запретить

### Данные заказчика

- Все данные принадлежат заказчику
- Полная выгрузка БД — `pg_dump` в любой момент
- Для Уровня 1: данные на вашем сервере, у нас нет доступа

## Мониторинг и резервное копирование

### Мониторинг (Уровни 2 и 3)

- systemd-сервисы под наблюдением
- Алерты в Telegram при сбоях
- Health-check эндпоинты:
  - `http://127.0.0.1:3000/health` — WhatsApp Bridge
  - `http://127.0.0.1:8099/health` — Document Extractor
- Prometheus + Grafana (Уровень 3)

### Резервное копирование (Уровни 2 и 3)

- `pg_dump` ежедневно, хранение 30 дней
- Еженедельный снапшот всего сервера (Уровень 3)
- Автоматическая ротация старых копий

### Для Уровня 1 (Self-hosted)

Резервное копирование и мониторинг — на вашей стороне. Рекомендуем настроить:
- cron: `pg_dump` раз в сутки
- systemd-сервисы с `Restart=always`

### Состав поставки (Уровень 1)

Вы получаете:

- Исходный код bot/ (лицензия MIT)
- SQL-схему базы данных (`db/ojr_schema.sql`)

- VPS-сервер
- API-ключи (Grok, OpenAI) — вы приобретаете самостоятельно
- SIM-карта и телефонный аппарат

### Сроки самостоятельного развёртывания

- Настройка сервера: 1–2 дня
- Развёртывание системой: 1–2 дня
- Подключение WhatsApp и Telegram: 1 день
- Тестирование в песочнице: 3–5 дней
- **Итого: 1–2 недели**
