# Технические требования для развёртывания Hermes — автономный AI-агент

**Продукт:** Hermes — AI-агент для управления строительными проектами через WhatsApp
**Дата:** 18.07.2026
**Статус:** Документ для заказчика

---

## Три уровня продукта

Hermes поставляется в трёх конфигурациях — от самостоятельной установки до полной интеграции в бизнес-процессы заказчика.

### Сравнительная таблица уровней

| | Уровень 1: Self-service | Уровень 2: С поддержкой | Уровень 3: Полная настройка |
|---|---|---|---|
| **Что получает клиент** | Репозиторий, документация, навыки Hermes | Всё из уровня 1 + настройка нашими специалистами | Всё из уровней 1+2 + интеграции и кастомные сценарии |
| **VPS** | Клиент предоставляет | Клиент предоставляет | Клиент предоставляет или наш VPS |
| **Настройка** | Самостоятельно по инструкции | Мы настраиваем удалённо | Мы настраиваем + кастомизируем |
| **Интеграции** | WhatsApp (базовая) | WhatsApp (базовая) | WhatsApp + Google Workspace + Bitrix24 + X/Twitter |
| **Обучение команды** | Документация | Сессия 2–3 часа | Расширенное обучение + поддержка |
| **Поддержка** | Community (чат/issues) | Реакция в течение 8 часов (рабочее время) | Реакция в течение 4 часов + выделенный менеджер |
| **Обновления** | Самостоятельно (Git pull) | Автоматически | Автоматически + backport кастомных изменений |
| **Срок запуска** | 1–3 дня (зависит от клиента) | 3–5 дней | 1–3 недели |

## Уровень 1 — Self-service

### Что входит

- Доступ к репозиторию Hermes (GitHub private repo)
- Документация по установке и настройке (`INSTALL.md`, `CONFIG.md`)
- Базовый набор навыков (skills) Hermes:
  - WhatsApp-мост (Hermes Bridge)
  - Ежедневный опрос (опросник строительного контроля)
  - Формирование ЕЖО (Excel)
  - Анализ фото через Grok Vision
  - Маршрутизация сообщений (Router)
  - Проверка погоды
  - QA-парсер (извлечение фактов из сообщений)
- Шаблоны конфигурационных файлов
- Скрипты инициализации БД (PostgreSQL)
- Доступ к community-чату поддержки

### Что нужно клиенту

| Позиция | Описание |
|---------|----------|
| **VPS** | Минимум 2 vCPU, 4 GB RAM, 20 GB SSD, Ubuntu 22.04/24.04 |
| **WhatsApp-номер** | Отдельный номер телефона для бота (не личный). SIM-карта, регистрация WhatsApp |
| **WhatsApp-группа «Песочница»** | Тестовая группа для отладки и тестирования |
| **WhatsApp-группа «Боевая»** | Рабочая группа стройки |
| **API-ключ xAI (Grok)** | Для AI-анализа сообщений и фото |
| **API-ключ OpenAI** | Для Codex CLI (само-улучшение кода) |
| **Данные объекта** | Название, адрес, заказчик, подрядчик, проектировщик, договор |
| **График СМР** | Этапы, даты начала/окончания (PDF/Excel) |
| **Приказы на ИТР** | Строительный контроль, авторский надзор |

### Ежемесячные расходы клиента (Уровень 1)

| Статья | ~Стоимость/мес |
|--------|---------------|
| VPS (2 vCPU, 4 GB) | ~$12–24 |
| xAI API (Grok) | ~$10–30 (зависит от объёма сообщений) |
| OpenAI API (Codex) | ~$20 (Pro) или pay-per-token |
| **Итого** | **~$42–74/мес** |

### Процесс самостоятельной установки

1. Клиент арендует VPS (2 vCPU, 4 GB RAM, Ubuntu 22.04/24.04)
2. Клиент получает доступ к репозиторию Hermes
3. Установка Docker и PostgreSQL по инструкции `INSTALL.md`
4. Клонирование репозитория, настройка `.env` (API-ключи, БД)
5. Запуск Hermes Bridge (WhatsApp)
6. Регистрация WhatsApp-номера через QR-код
7. Добавление бота в группы (песочница → тестирование → боевая)
8. Инициализация БД ОЖР, загрузка данных объекта и графика СМР

## Уровень 2 — С поддержкой

### Что входит (всё из уровня 1 +)

- Pre-configured Hermes с оптимизированными настройками
- Полный комплект навыков Alikhan:
  - ЕЖО (шаблон Excel + авто-заполнение)
  - Ежедневный опрос (логика фильтрации, запуск, статус)
  - Месячный шаблон (подготовка следующего периода)
  - Анализ фото строительной площадки (Grok Vision)
  - Суточный снимок дня (сбор фото, сообщений, погоды)
- Настроенные cron-задачи:
  - Ежедневный опрос (автоматически по расписанию)
  - Ежедневный отчёт (сбор данных за день)
  - Ежемесячный шаблон (формирование следующего периода)
  - Бэкап БД
- Удалённая настройка нашими специалистами
- Обучение команды: сессия 2–3 часа (Zoom/Google Meet)
  - Как работает бот
  - Как отвечать на опрос
  - Как читать ЕЖО
  - Как добавлять данные
- Мониторинг работоспособности (uptime-проверки)
- Автоматические обновления из репозитория

## Безопасность

### WhatsApp

- **Две группы:** песочница (тестовая) + боевая (рабочая)
- Бот НЕ пишет в боевую группу без явной команды
- Все изменения тестируются в песочнице перед выкатом в боевую
- Номер бота — отдельный, не привязанный к личным устройствам

### Управление Hermes

- **Telegram / Discord** — закрытые каналы для административных команд
- Hermes слушает команды только от авторизованных пользователей
- Логирование всех административных действий

### Данные

- **Вариант A:** VPS клиента — данные полностью под контролем заказчика
- **Вариант B:** Наш VPS — данные изолированы в отдельном проекте, ежедневные бэкапы
- API-ключи хранятся в `.env` (не коммитятся в репозиторий)
- PostgreSQL с парольной защитой (не слушает внешние порты)
- Бэкапы БД — ежедневно (уровни 2–3)

## Часто задаваемые вопросы

### «Почему WhatsApp, а не Telegram?»

WhatsApp — стандарт деловой коммуникации на стройках Кыргызстана и СНГ. Прорабы, ИТР, субподрядчики — все уже там. Не нужно никого переучивать. Telegram используется для административного управления Hermes (закрытый канал).

### «Нужен ли постоянный доступ в интернет на объекте?»

Боту — да (VPS). Пользователям в WhatsApp — обычный мобильный интернет, как всегда.

### «Что если API-ключ закончится?»

Hermes пришлёт предупреждение администратору в Telegram/Discord. Grace period — 24 часа. Критические функции (опрос, погода) продолжают работать без AI.

### «Можно ли перенести данные с другого объекта?»

Да. База ОЖР — стандартный PostgreSQL. Можно выгрузить, передать заказчику, приобщить к исполнительной документации.

### «Куда сохраняются фото?»

В БД (`ojr_photo_log`) с привязкой к дате и зданию. Каждое фото получает текстовое описание от Grok Vision.

### «Можно ли перейти с уровня на уровень?»

Да. Уровень 1 → Уровень 2: доплата за настройку и обучение. Уровень 2 → Уровень 3: доплата за интеграции и кастомизацию. Данные сохраняются.

### «Что делать если VPS упал?»

- Уровень 1: перезапустить VPS, проверить systemd-сервисы (`systemctl --user status hermes-whatsapp-bridge`)
- Уровень 2–3: мы мониторим и восстанавливаем (реакция 4–8 часов в рабочее время)
