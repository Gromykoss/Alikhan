# Симуляция ЕЖО 28.06.2026

Полная методика, архитектура цикла, все правки кода и результаты проверки.

## Цикл ЕЖО

```
QA-опрос (вечер) → QA-факты в PostgreSQL
                         ↓
          fill_ejo.py (авто-генерация)
                         ↓
          /tmp/ЕЖО_2026-06-28_vN.xlsx
                         ↓
          Ручная правка пользователем
                         ↓
          Отправка в песочницу WhatsApp
                         ↓
          main_waha.py: сравнение с авто, сохранение шаблона
                         ↓
          templates/ЕЖО_шаблон.xlsx + /tmp/ЕЖО_2026-06-28_v1.xlsx
                         ↓
          Следующий день: генерация из обновлённого шаблона
```

## Методика симуляции

1. Загрузка исторических QA-фактов за 27.06 и 28.06 в PostgreSQL
2. Авто-генерация 28.06: `python3 fill_ejo.py 2026-06-28`
3. Ручная правка: добавление кодов 4-го уровня, исправление накопленных
4. Отправка правленого файла: бот принимает, сравнивает с авто, обновляет шаблон
5. Повторная генерация: проверка отсутствия удвоения

## Ключевые правки

1. **Персонал по профессиям:** `get_aibikon_headcount()` → `{'total': N, 'by_prof': {...}}`, колонка C табеля
2. **Сравнение по кодам:** `build_code_map()` — исключает ложные diff при вставке строк
3. **Накопленные:** `v>0 → yesterday_cum()`, шаблон только для кодов без сегодняшних работ
4. **data_only=True:** загрузка шаблона с кешированными значениями
5. **Дата из имени файла:** `re.search(r'(\d{2})\.(\d{2})\.(\d{4})', fname)`
6. **Сортировка по mtime:** `key=os.path.getmtime`
7. **Логотип:** удалён из кода (уже в шаблоне)
8. **Bootstrapping:** первый цикл требует исправленный файл за предыдущий день
