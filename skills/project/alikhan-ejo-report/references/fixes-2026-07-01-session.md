# Fixes 2026-07-01 — боевая сессия (v2.8.0)

## 1. SIM_DATE рассинхрон router.py ↔ main_waha.py

**Баг:** `router.py:9` имел `SIM_DATE = "2026-06-30"`, тогда как `main_waha.py:15` уже `SIM_DATE = None`. При вызове `parse_qa(date_str=SIM_DATE)` из router, все QA-факты сохранялись с датой 2026-06-30 независимо от реальной даты.

**Симптом:** факты за 01.07 сохранялись как 30.06, fill_ejo не видел их, опросник показывал «нет данных».

**Фикс:** `router.py:9` — `SIM_DATE = None`.

**Правило:** при смене SIM_DATE менять в ОБОИХ файлах одновременно:
- `main_waha.py:15` (основной цикл)
- `router.py:9` (роутер → parse_qa)

## 2. Команда «формируй отчет» не распознавалась

**Баг:** пользователь пишет «Алихан формируй отчет». `router.py:30` имел только `"формируй ежо"`, но не `"формируй отчет"`. Бот отвечал «Нужно уточнить в БД. Какой именно отчет?».

**Фикс — router.py:30:** добавлены `"формируй отчет"`, `"сформируй отчет"`, `"сделай отчет"`, `"заполни отчет"`.

**Фикс — main_waha.py:349:** тот же список команд — без этого CMD из роутера не запускал fill_ejo.

## 3. Планы на завтра: поиск только в бетонирование

**Баг:** `smart_evening_check.py:81` — `WHERE ... AND category='бетонирование'`. QA-факт «План на завтра 2.4.2.2 = 2000 кг» сохранён в категории `монтаж` → не найден → опросник: «❌ Планы на завтра — нет».

**Фикс:** убрать фильтр по категории — `WHERE ... AND fact ILIKE '%%план%%'` по ВСЕМ категориям.

## 4. Материалы не авто-заполнялись при закрытии опроса

**Баг:** `main_waha.py:316-318` — `cats` dict содержал только бетонирование/монтаж/земляные работы. Категория `документация` (куда сохраняются материалы) не авто-заполнялась → опросник: «❌ Материалы — нет данных».

**Фикс:** добавлена `'документация': 'Поставок материалов нет'` в `cats`.

## 5. «опрос окончен» не в триггерах закрытия

**Баг:** `main_waha.py:309` — список триггеров закрытия опроса не содержал «опрос окончен». Пользователь написал «опрос окончен» → бот не закрыл опрос.

**Фикс:** добавлены `"опрос окончен"` и `"опрос завершен"`.

## 6. Жёлтые ячейки E12-G12 (происшествия, строка 12)

**Баг:** `fill_ejo.py` swaps dict содержал E11/G11 но не E12/G12. После заполнения ячейки [12,5]-[12,7] оставались жёлтыми со значением 0.

**Фикс:** добавлены `'E12': inc, 'F12': inc, 'G12': inc` в swaps.
