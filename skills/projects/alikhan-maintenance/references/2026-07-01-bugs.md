## 2026-07-01: Four Bugs Found During Simulation Testing

### Bug 1: Router sends commands through Grok→verify

**Symptom:** `[VERIFY] REJECT 35: Не сообщил об отсутствии данных в БД, вместо этого сразу запустил опрос.`

**Root cause:** `router.py` treats "Алихан запускай опрос" as a question. `lookup_facts()` returns None → falls to Grok fallback → generates reply → verify scores it 35.

**Fix:** Added `cmd_words` list in `router.py` (line ~28-34) — returns `CMD` action early, skipping Grok/verify. Added "CMD" to skip list in `main_waha.py` (line ~374).

### Bug 2: 10-minute message timeout

**Symptom:** User's messages appear in log as `[MSG]` but never get processed — no `[REPLY]` follows.

**Root cause:** `main_waha.py` line 188: `if now_ts - msg_ts > 600: continue` — messages older than 10 minutes are silently skipped. During debugging, restarts take >10 min, all test messages age out.

**Fix:** Removed the timeout entirely. The `seen` set already handles deduplication.

### Bug 3: Seed adds user messages to seen set

**Symptom:** After restart, bot seeds N recent messages as "seen" — including the user's test messages. They never get processed because `if mid in seen: continue`.

**Root cause:** `main_waha.py` lines 148-154: seed loop adds ALL recent messages to `seen`, regardless of `fromMe`.

**Fix:** Seed only adds messages where `m["key"].get("fromMe")` is true (bot's own messages). User messages flow to the main loop for processing.

### Bug 4: Command word mismatch

**Symptom:** Router returns `CMD` (correct), but the survey trigger in `main_waha.py` never fires.

**Root cause:** Router has "запускай опрос" in `cmd_words`, but the survey trigger checks for "запустить опрос" — different verb form. User types "запускай", code expects "запустить".

**Fix:** Added "запускай опрос" to the survey trigger word list (line 332).
