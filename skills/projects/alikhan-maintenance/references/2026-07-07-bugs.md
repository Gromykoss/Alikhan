## 2026-07-07: Four Bugs Found and Fixed

### Bug 1: "Работы выполнены" ignored — name check kills VOR codes

**Symptom:** Foreman writes something like "Работы выполнены 2.4.2 — 104.3 м3" — bot stays silent. Log shows `[MSG]` but `action == "IGNORE"`, no `[REPLY]`.

**Root cause:** `router.py` uses regex `[ао]л[еи][хгк]` to require "алихан" in every message. Messages with VOR codes but without "алихан" (e.g., plain updates from the foreman) are silently dropped.

**Fix (two files):**
1. **router.py** (line 25-27): Added work-code regex check BEFORE the name check:
```python
if re.search(r'\d+\.\d+\.\d+(?:\.\d+)?\s*[=—–\-:\s]+\s*\d+', text):
    return "RESIDUAL", "", False
```
Returns `"RESIDUAL"` action instead of `"IGNORE"`.

2. **main_waha.py** (lines 527-536): Added `action == "RESIDUAL"` handler after `route()`:
```python
if action == "RESIDUAL":
    from poll import parse_poll_reply
    today_str = SIM_DATE or datetime.now().strftime("%Y-%m-%d")
    result = parse_poll_reply(text, SANDBOX, today_str)
    if result.get('codes_updated'):
        reply = f"✅ Принято: {len(result['codes_updated'])} кодов"
        for c in result['codes_updated']:
            reply += f"\n  • `{c['code']}`: {c['actual_today']}"
        send_msg(SANDBOX, reply)
    continue
```

**Verification:** Send `"2.4.2 — 104.3"` to sandbox group. Bot should reply "✅ Принято: 1 кодов" without requiring "алихан" in the message.

### Bug 2: Section 3.1 missing from ACTIVE_SECTIONS

**Symptom:** Work items in section 3.1 (e.g., 3.1.1, 3.1.2, 3.1.3) never appear in polls. `_get_work_items_from_template()` returns 0 items even when the template has valid data with `ostatok > 0`.

**Root cause:** `poll.py:88`:
```python
ACTIVE_SECTIONS = {"2.1", "2.2", "2.3", "2.4", "3.2", "3.3"}
```
`"3.1"` was missing. The filter at line 110 drops any code whose first two segments aren't in the set.

**Fix:** Added `"3.1"`:
```python
ACTIVE_SECTIONS = {"2.1", "2.2", "2.3", "2.4", "3.1", "3.2", "3.3"}
```

**Detection:** When poll has 0 work items, add print debugging at `poll.py:110`:
```python
print(f"[POLL] Section {section} {'IN' if section in ACTIVE_SECTIONS else 'OUT'}")
```

### Bug 3: Template comparison only checks 3 of 7 data columns

**Symptom:** User corrects the ЕЖО template (modifying monthly plan, monthly %, SMR cumulative, SMR % columns). Bot replies "📎 Правки приняты (0 отличий)". Changes go undetected.

**Root cause:** `main_waha.py:_update_template_from_correction()` — `build_code_map()` only captured cols 16, 19, 21, and the comparison loop only checked those 3:
```python
cmap[code] = {'row': r, 16: ..., 19: ..., 21: ...}
for col, name in [(16, 'мес.факт'), (19, 'общ.факт'), (21, 'остаток')]:
```

**Fix:** Expand to all 7 columns (O-U = cols 15-21):

`build_code_map()` now captures:
```python
cmap[code] = {
    'row': r,
    15: ws.cell(r, 15).value,   # мес.план
    16: ws.cell(r, 16).value,   # мес.факт
    17: ws.cell(r, 17).value,   # мес.%
    18: ws.cell(r, 18).value,   # общ.план
    19: ws.cell(r, 19).value,   # общ.факт
    20: ws.cell(r, 20).value,   # общ.%
    21: ws.cell(r, 21).value,   # остаток
}
```

Comparison loop checks all 7:
```python
for col, name in [(15, 'мес.план'), (16, 'мес.факт'), (17, 'мес.%'),
                   (18, 'общ.план'), (19, 'общ.факт'), (20, 'общ.%'), (21, 'остаток')]:
```

**Limitation:** Cell fill (ззаливка), photos (лист «Фототчет»), and text changes on other sheets are still not compared. Only 7 numeric columns on the «Ежедневный отчет» sheet.

### Bug 4: `int(None)` crash in poll.py format strings

**Symptom:** `build_poll_message()` or `build_poll_summary()` raises:
```
TypeError: int() argument must be a string, a bytes-like object or a real number, not 'NoneType'
```
Crashes mid-poll when formatting output.

**Root cause:** Three locations where `int()` is called on DB-or-template values that can be `None`:

1. **poll.py L280** (in `build_poll_message`):
```python
o = f"{item['ostatok']:.0f}" if item['ostatok'] == int(item['ostatok']) else f"{item['ostatok']:.1f}"
```
`item['ostatok']` comes from template row — `_safe_float` returns 0 for None, but if the template-read generates a different code path where the value is None, this crashes.

2. **poll.py L470** (in `build_poll_summary`):
```python
o = f"{r['residual_volume']:.0f}" if r['residual_volume'] == int(r['residual_volume']) else ...
```
`residual_volume` is DB `NUMERIC` — can be NULL for incomplete poll residuals.

3. **poll.py L471**:
```python
a = f"{r['actual_today']:.0f}" if r['actual_today'] == int(r['actual_today']) else ...
```
`actual_today` is DB `NUMERIC` — can be NULL if not yet updated by foreman reply.

**Fix:** Added `is not None` guards at all three locations with `else '0'` fallback:

```python
# Line 280
o = f"{item['ostatok']:.0f}" if item['ostatok'] and item['ostatok'] == int(item['ostatok']) else f"{item['ostatok']:.1f}" if item['ostatok'] else '0'

# Lines 470-471
o = f"{r['residual_volume']:.0f}" if r['residual_volume'] is not None and r['residual_volume'] == int(r['residual_volume']) else f"{r['residual_volume']:.1f}" if r['residual_volume'] is not None else '0'
a = f"{r['actual_today']:.0f}" if r['actual_today'] is not None and r['actual_today'] == int(r['actual_today']) else f"{r['actual_today']:.1f}" if r['actual_today'] is not None else '0'
```

**Verification:** Run `close_poll()` on a poll where some residuals have NULL `actual_today`. The summary should show `0` instead of crashing.
