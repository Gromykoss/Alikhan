## Источник истины

Шаблон `ЕЖО_шаблон.xlsx`, лист «Ежедневный отчет». Два ключевых столбца:

| Столбец | Колонка | Назначение |
|---------|---------|------------|
| O | 15 | План на месяц — фильтр опроса |
| U | 21 | Остаток на месяц — для отображения |

## Месячный цикл

### Начало месяца — план
1. «Алихан раскрой отчет» → бот присылает шаблон со всеми строками
2. Пользователь заполняет O (план) и U (остаток) → присылает файл обратно
3. Бот сохраняет шаблон. **Опрос НЕ закрывается.**

### Ежедневный опрос
1. «Алихан запускай опрос» → бот: **O > 0 ∧ U > 0**
2. QA-сводка + список работ с остатками (все здания из колонки A)
3. Прорабы: `код = объём` (только где были работы сегодня)
4. «Алихан закончить опрос» или «Алихан формируй отчет» → ЕЖО

## Правила фильтра

```python
if monthly_plan <= 0:      # col O — не в плане
    continue
if ostatok <= 0:           # col U — нет остатка
    continue
```

- Никаких schedule-based фильтров
- Никаких вычислений из plan_cum - fact_cum
- Все здания динамически (for bld in by_bld)

## Команды

| Команда | Действие |
|---------|----------|
| `Алихан раскрой отчет` | Шаблон со всеми строками |
| `Алихан запускай опрос` | Опрос O>0 ∧ U>0 |
| `Алихан статус опроса` | Сводка (без «не хватает остатков») |
| `Алихан формируй отчет` | Закрыть опрос → ЕЖО (без дубля сводки) |
| `Алихан закончить опрос` | То же самое |

## ⛔ Pitfalls

### P0. Три этапа — не мешать (16.07.2026)

**Месячный план, опрос, и формирование ЕЖО — три РАЗНЫХ этапа.**
- План на месяц: все строки видны, O и U вручную.
- Опрос: O>0 ∧ U>0, частичные данные — нормально.
- ЕЖО: формулы fill_ejo.py, скрытие строк без работ.
НЕ переносить правила между этапами. Пользователь: «не нужно мешать все в кучу».

### P1. Не менять код когда данные не готовы

Если фаза не попала в опрос — O или U не заполнены. Сказать пользователю что заполнить.
НЕ менять фильтр на вычисления. Пользователь: «ты должен сказать что я нарушил логику, а не менять логику туда-сюда».

### P2. Шаблон НЕ закрывает опрос (AL-015)

`_update_template_from_correction()` авто-закрывал опрос. Убрано 16.07.2026.
Шаблон обновляется — опрос живёт своей жизнью.

### P3. Остаток месячный ≠ сегодняшний объём

U — на месяц. Работы сегодня — только где реально делали. Не все 28 кодов каждый день.
- Статус НЕ флажит «⚠️ Не хватает остатков»
- Закрытие НЕ требует остатков (только персонал + техника)
- Сообщение: «сегодня X (остаток Y)»

### P4. HTTP 413 — Express body limit (AL-016)

ЕЖО ~330KB base64 = ~440KB. Express default 100KB → PayloadTooLargeError.
Фикс: `app.use(express.json({limit: '50mb'}))` в bridge.js.

### P5. sendMedia через bridge_wrapper

`_send_document` использует `requests.post` → bridge_wrapper перехватывает
`/message/sendMedia/` → b64decode → tempfile → bridge `/send-media`.
Раньше был только urllib-патч, `requests.post` падал с 404.

### P6. Перезаливка документа — local_path (AL-022)

Старые документы из Evolution API не имеют `local_path` в tags.
При перезаливке бот считает дубликатом (content совпадает) и пропускает.
Фикс: `UPDATE tags = tags || '{"local_path":"..."}'::jsonb WHERE content = fname`.
Табель читается через `get_aibikon_headcount()` из локального кеша, не через bridge API.
