## Формат отчёта (v3 — 10.07.2026)

**3 листа:** «Ежедневный отчет», «Персонал и техника», «Материалы и планы». Лист «Фототчет» **удалён**.

### Фотоотчёт (строки 856-859)

Фото вставляются **горизонтально** с шагом +2 колонки на каждое последующее фото:
- A856 = «Общежитие»: фото1→B856, фото2→D856, фото3→F856
- A857 = «АБК»: фото1→C857, фото2→E857, фото3→G857
- A858 = «Галерея»: фото1→D858, фото2→F858, фото3→H858
- A859 = «Общие планы»: фото1→E859, фото2→G859, фото3→I859

**КРИТИЧНО: перед вставкой фото — `ws._images.clear()`.** Без этого старые фото из шаблона остаются и наслаиваются на новые.

### Готовность объекта (строка 853)

- **J853** — очистить (убрать «x%» из шаблона): `sw(ws, 853, 10, None)`
- **K853** — `f"{pct}%"` с жёлтой заливкой (FFFF00): `ws.cell(853, 11).fill = yellow_fill`
- **Формула:** `round(base_pct × 0.94 + 6)` где base_pct = `sum(plan × rate) / sum(plan)` по всем строкам разделов 2-8
- Раздел 1 (ПСД) = 6% от проекта, всегда 100% (в шаблоне нет строк раздела 1 — добавляется константой)

## Цепочка накопов (yesterday_cum)

```python
def yesterday_cum(date, code):
    yd = date - timedelta(days=1)
    for v in range(10, 0, -1):  # ищем v10 → v1
        p = f"/tmp/ЕЖО_{yd}_v{v}.xlsx"
        if os.path.exists(p):
            # Читаем колонку 16 (месячный накоп) и 19 (СМР накоп)
            pm = ws.cell(r, 16).value  # P — накоп за месяц
            pt = ws.cell(r, 19).value  # S — накоп с начала СМР
            return (float(pm), float(pt))
    return (0, 0)  # если файла нет → с нуля
```

**Ключевое:** накопы НЕ хранятся в БД. Они живут только в Excel-файлах. Каждый день читается вчерашний файл.

**v1 vs v2+:** если `v > 0` (работа сегодня) → `prev = yesterday_cum()`. Если `v == 0` (нет работы) → `prev = max(шаблон, yesterday_cum)`.

## Алгоритм fill() для работ

```
1. Загружаем шаблон (data_only=True)
2. vols = volumes(date) — сегодняшние объёмы из QA
3. Очищаем daily columns [12,13,14] для ВСЕХ строк с VOR-кодами
4. Для строк с работой сегодня (code in vols): очищаем [16,19,21]
5. Для строк с работой сегодня:
   a. prev_p, prev_s = yesterday_cum(date, code)
   b. L=12, M=13 = v (факт за сутки)
   c. N=14 = 1
   d. P=16 = prev_p + v (накоп за месяц)
   e. Q=17 = (prev_p+v) / mp (% за месяц)
   f. S=19 = prev_s + v (накоп СМР)
   g. T=20 = (prev_s+v) / tp (% СМР)
   h. U=21 = k_plan - prev_s - v (остаток)
   i. Заливка жёлтым ВСЕЙ строки A-U, **только если L=12 или M=13 > 0** (есть объём сегодня)
```

### Жёлтая заливка — правило (v3, фикс 10.07.2026)

**Правило:** строка красится жёлтым (FFFF00) по колонкам A-U ТОЛЬКО если в ней заполнены суточные объёмы (L=12 план или M=13 факт > 0).

**Антипаттерн (исправлено):** красить все строки активных фаз жёлтым. Фаза может быть «в работе» по графику, но если сегодня нет объёмов — строка остаётся белой. Жёлтый = «сегодня работали», а не «фаза активна».

```python
plan_val = ws.cell(r, 12).value  # L — план за сутки
fact_val = ws.cell(r, 13).value  # M — факт за сутки
has_volume = False
try:
    if plan_val is not None and float(plan_val) > 0:
        has_volume = True
    if fact_val is not None and float(fact_val) > 0:
        has_volume = True
except (ValueError, TypeError):
    pass
if has_volume:
    yellow_fill = PatternFill(start_color=Color(rgb='FFFF00'), end_color=Color(rgb='FFFF00'), fill_type='solid')
    for c in range(1, 22):
        ws.cell(r, c).fill = yellow_fill
```

## Guard logic (защита правок)

1. **Daily columns (12,13,14):** очищаются ВСЕГДА для всех строк с кодами. Старые суточные данные из шаблона не «застывают».
2. **Cumulative columns (16,17,19,20,21):** очищаются ТОЛЬКО для строк с работой сегодня → пересчитываются через yesterday_cum(). Для строк БЕЗ работы накопы СОХРАНЯЮТСЯ (ручная правка не теряется).
3. **Template update:** пользователь присылает исправленный ЕЖО → `_update_template_from_correction()` → сравнение diff по ВСЕМ колонкам → исправленный файл становится новым шаблоном + сохраняется как `ЕЖО_{date}_v1.xlsx`.

## Колонки (строка 20 — заголовки)

| Col | Letter | Excel-заголовок | Код пишет | Формула |
|-----|--------|----------------|-----------|---------|
| 11 | K | Кол-во | — | План (контракт) |
| 12 | L | Объем за сутки | `v` | Факт сегодня |
| 13 | M | (merged) | `v` | Дубль L |
| 14 | N | (merged) | `1` | Дней работы |
| 15 | O | Накопительный за месяц | — | Заголовок (merged) |
| 16 | P | — | `prev_p + v` | Месячный накоп |
| 17 | Q | — | `(prev_p+v)/mp` | % за месяц |
| 18 | R | Накопительный с начала СМР | — | Заголовок (merged) |
| 19 | S | — | `prev_s + v` | СМР накоп |
| 20 | T | — | `(prev_s+v)/tp` | % СМР |
| 21 | U | — | `k_plan−prev_s−v` | Остаток |

## Pitfall: merged cells

Заголовки «Накопительный за месяц» (O=15) и «Накопительный с начала СМР» (R=18) — это merged cells. Значения пишутся в правые подколонки: P=16 и S=19. `openpyxl` с `data_only=True` может вернуть None для merged cells. Всегда читай значения из колонок 16 и 19, а не 15 и 18.
