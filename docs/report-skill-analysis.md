## 1. Текущая архитектура отчётов

```

         → Switch Command (14 веток)
              ├─ memory_status          → PostgreSQL → Set → WhatsApp
              ├─ group_participants     → PostgreSQL → Set → WhatsApp
              ├─ participant_activity   → PostgreSQL → Set → WhatsApp
              ├─ period_summary         → PostgreSQL → xAI → Set → WhatsApp
              ├─ fact_lookup            → PostgreSQL → xAI → Set → WhatsApp
              ├─ quoted_document_summary→ PostgreSQL → xAI → Set → WhatsApp
              ├─ id_lookup              → PostgreSQL → Set → WhatsApp
              ├─ document_compare       → PostgreSQL → Loop xAI → WhatsApp
              ├─ calendar_list          → PostgreSQL → Set → WhatsApp
              ├─ calendar_create        → xAI → Parse → PostgreSQL → WhatsApp
              ├─ calendar_delete        → PostgreSQL → Set → WhatsApp
              ├─ current_datetime       → PostgreSQL → Set → WhatsApp
              ├─ only_name              → Static → WhatsApp
              ├─ who_are_you            → Static → WhatsApp
```

## 2. Детальный разбор существующих отчётов

### 2.1. `memory_status` — Статистика памяти

**Триггер:** `алихан memory_status`  
**Данные:** PostgreSQL `bot_memory_messages`  
**SQL:**
```sql
SELECT
  COUNT(*) FILTER (WHERE chat_id = 'project:main' AND message_type = 'text') AS project_text_messages,
  COUNT(*) FILTER (WHERE chat_id = 'project:main' AND message_type = 'image') AS project_image_messages,
  COUNT(*) FILTER (WHERE chat_id = 'project:main' AND message_type = 'image' AND content ILIKE '%Ошибка%') AS project_image_errors,
  COUNT(*) FILTER (WHERE chat_id = '{current_chat}') AS current_chat_messages
FROM bot_memory_messages;
```

**Формат вывода:** Одна строка с цифрами.  
**Проблема:** нет визуализации, нет трендов, нет разбивки по дням.

### 2.2. `group_participants` — Участники группы

**Триггер:** `алихан участники` / NLP-роутинг  
**Данные:** WhatsApp API через evolution-api → n8n Code node парсит  
**Формат вывода:** Список имён + номеров.  
**Проблема:** статичный снимок, нет истории изменений состава.

### 2.3. `participant_activity` — Активность участников

**Триггер:** `алихан активность` / NLP  
**Данные:** PostgreSQL (агрегация по `bot_memory_messages.sender`)  
**Формат вывода:** Топ-N по количеству сообщений.  
**Проблема:** нет фильтра по датам, нет типов сообщений (текст/фото/документ).

### 2.4. `period_summary` — Периодный отчёт

**Триггер:** `алихан итоги за неделю` / NLP с парсингом дат  
**Данные:** PostgreSQL → xAI  
**Формат вывода:** AI-сгенерированный текст.  
**Сильная сторона:** гибкий NLP-парсинг дат (русские месяцы, диапазоны).  
**Проблема:** нет структуры, нет секций, нет ключевых метрик.

### 2.5. `fact_lookup` — Поиск фактов

**Триггер:** `алихан найди про демонтаж` / NLP  
**Данные:** PostgreSQL (полнотекстовый поиск) → xAI  
**Формат вывода:** AI-пересказ найденного.  
**Проблема:** нет ссылок на исходные сообщения, нет confidence score.

### 2.6. `document_compare` — Сравнение документов

**Триггер:** `алихан сравни документы` / NLP  
**Данные:** PostgreSQL (поиск документов) → Loop xAI → Aggregation  
**Формат вывода:** AI-сравнение.  
**Сильная сторона:** самая сложная логика — чанкинг, агрегация, Wait-ноды.  
**Проблема:** медленно (много API-вызовов xAI), нет кэширования.

### 2.7. Календарные отчёты

- `calendar_list` — список событий
- `calendar_create` — создание через NLP
- `calendar_delete` — удаление
- Напоминания: отдельный workflow `Calendar Reminders` (каждую минуту)

## 3. Проблемы текущей архитектуры

### 3.1. Архитектурные

| Проблема | Влияние |
|----------|---------|
| 82 ноды в одном workflow | Невозможно отлаживать, править, деплоить |
| Отчёты зашиты в монолит | Нельзя переиспользовать логику отчётов |
| Нет версионирования отчётов | Не видно историю изменений форматов |
| SQL запросы вшиты в JSON | Невозможно code review SQL |
| Нет обработки ошибок | При падении PostgreSQL — тишина |

### 3.2. Функциональные

| Проблема | Что нужно |
|----------|-----------|
| Отчёты — стена текста | Структура: секции, метрики, выводы |
| Нет scheduled reports | Автоматические еженедельные/ежемесячные |
| Нет экспорта | PDF, DOCX, Markdown |
| Нет дашборда | Ключевые метрики в одном сообщении |
| Нет алертов | Критические изменения → уведомление |
| Один chat_id | Отчёт только в ту же группу, нет доставки в другие каналы |

### 3.3. UX

| Проблема | Что нужно |
|----------|-----------|
| Надо помнить команды | Меню, `/report` с подсказками |
| Нет preview | Не видно что будет в отчёте до генерации |
| Длинные ответы режутся | Пагинация, разбивка на части |
| Нет feedback loop | Нельзя уточнить/перегенерировать отчёт |

## 4. Целевая архитектура: Report Skill

### 4.1. Принципы

1. **Модульность** — каждый тип отчёта = отдельный модуль
2. **Структурированный вывод** — секции, таблицы, метрики, выводы
3. **Scheduled + On-demand** — и автоматические, и по запросу
4. **Множественная доставка** — WhatsApp, Telegram, файл
5. **Версионирование** — шаблоны отчётов в Git
6. **AI-first** — xAI для анализа, SQL для агрегации

### 4.2. Типы отчётов (целевые)

#### А. Быстрые (в реальном времени, <5 сек)

| Отчёт | Триггер | Данные | Формат |
|-------|---------|--------|--------|
| `/r status` | Команда | PostgreSQL counts | Секции: чат, фото, документы, календарь |
| `/r today` | Команда | Сообщения за сегодня | Кол-во, топ участников, темы |
| `/r week` | Команда | Сообщения за 7 дней | Тренды, график активности, ключевые события |
| `/r calendar` | Команда | Предстоящие 7 дней | Список с датами и временем |

#### Б. AI-отчёты (10-30 сек, xAI)

| Отчёт | Триггер | Данные | Формат |
|-------|---------|--------|--------|
| `/r summary` | Команда | Период + все данные | AI-сводка со структурой |
| `/r project` | Команда | Все данные проекта | Статус, прогресс, риски |
| `/r person ИМЯ` | Команда | Сообщения человека | Профиль активности, темы, документы |

#### В. Scheduled (cron)

| Отчёт | Расписание | Доставка |
|-------|------------|----------|
| Daily Digest | 08:00 | WhatsApp группа |
| Weekly Report | Пн 09:00 | WhatsApp + Telegram |
| Monthly Report | 1-е число 10:00 | WhatsApp + файл |
