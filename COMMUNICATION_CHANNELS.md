# Каналы связи Hermes: варианты для заказчика

**Дата:** 18.07.2026
**Статус:** Документ для заказчика

---

## Два типа каналов

У каждого объекта Alikhan два раздельных канала связи:

| Канал | Назначение | Кто общается |
|-------|-----------|-------------|
| **WhatsApp** | Сбор данных со стройки | Бот ↔ прорабы, ИТР, субподрядчики |
| **Второй канал** | Управление Hermes, настройка, обслуживание | Администратор ↔ Hermes |

**WhatsApp — не обсуждается.** Это основной сбор данных, другого варианта для стройплощадки нет.

**Второй канал** — можно выбрать:

## Второй канал: управление Hermes

Через этот канал администратор:
- Отдаёт команды Hermes («сделай снимок дня», «сформируй ЕЖО», «покажи логи»)
- Получает системные уведомления (ошибки, алерты, статус)
- Настраивает параметры бота
- Даёт approval на отправку в боевую группу
- Управляет обслуживанием и расширением функционала

### Варианты

| Канал | Статус | Удобство | Что нужно |
|-------|--------|----------|-----------|
| **Telegram** | ✅ Готово | ⭐⭐⭐⭐⭐ | Создать бота через @BotFather |
| **Discord** | ✅ Готово | ⭐⭐⭐ | Создать сервер + бота |

## 1. Telegram — рекомендованный

### Преимущества
- Бесплатный, не нужен отдельный номер
- Мгновенные уведомления на телефон
- Поддержка Markdown, файлов, inline-кнопок
- Hermes уже интегрирован «из коробки»

### Что нужно

| Позиция | Кто делает |
|---------|-----------|
| Создать бота через @BotFather (2 минуты) | Мы или заказчик |
| Токен бота | Выдаёт BotFather |
| Подписаться на бота в Telegram | Заказчик |

### Процесс
1. Создаём бота → получаем токен
2. Прописываем токен в Hermes
3. Заказчик открывает чат с ботом
4. Hermes отвечает на команды, шлёт уведомления

## WhatsApp — основной сбор данных

WhatsApp **не обсуждается как опция** — это единственный канал сбора данных со стройплощадки.

### Почему
- 100% охват на стройках СНГ и Кыргызстана
- Прорабы и ИТР уже там — не нужно ставить новые приложения
- Групповые чаты — стандартный формат коммуникации на объекте
- Фото, документы, голосовые сообщения — всё в одном месте

### Что нужно

| Позиция | Кто делает |
|---------|-----------|
| SIM-карта для бота | Заказчик покупает |
| Группа «Песочница» | Заказчик создаёт |
| Группа «Боевая» | Заказчик создаёт |
| Hermes Bridge (WhatsApp Web API) | Мы настраиваем |

### Процесс
1. Заказчик покупает SIM, регистрирует WhatsApp
2. Мы генерируем QR-код на VPS
3. Заказчик сканирует QR через WhatsApp → Linked Devices
4. Бот онлайн 24/7

## Итоговая схема

```
Стройплощадка                Администратор
     │                             │
     │ WhatsApp                    │ Telegram / Discord / Slack
     ▼                             ▼
┌─────────┐                 ┌─────────┐
│ Alikhan │◄───────────────►│ Hermes  │
│  (бот)  │   управление    │ (агент) │
└────┬────┘                 └────┬────┘
     │                           │
     │                           │ Codex / Grok Build
     ▼                           ▼
  ОЖР БД                   Само-улучшение кода
  ЕЖО
  Снимок дня
```

**WhatsApp** — данные текут в одну сторону (со стройки → в БД), бот отвечает только в песочнице.
**Второй канал** — двусторонняя связь: администратор даёт команды, Hermes отвечает.
