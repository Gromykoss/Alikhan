# METHODOLOGY — Методология работы с ботом Алихан

## Опрос остатков работ (poll)

### Команды

| Команда | Действие |
|---------|----------|
| «Алихан начать опрос» / «запустить опрос» | Создать опрос, проверить собранные данные, запросить недостающее |
| «Алихан закончить опрос» / «завершить опрос» | Закрыть опрос, авто-заполнить пропуски, сформировать ЕЖО |
| «Алихан статус опроса» / «что собрано» | Показать сводку собранных/недостающих данных |
| «Алихан заполни ежо» / «сформируй ежо» | Проверить опрос и сформировать ЕЖО (если всё собрано) |

### Формат ответа прораба

**Остатки работ (VOR-коды):**
```
2.1.1 = 85.5
2.3.2 = 50
2.4.1 — 120м3
```

**Персонал:**
```
Атантай ИТР 2, рабочих 15
Наватек: ИТР 1, рабочих 8
```

**Техника:**
```
Экскаватор 1, самосвал 2, автокран 1
```

**Материалы:**
```
Поставок материалов нет
Поставка арматуры 5 тонн
```

**Планы на завтра:**
```
План на завтра: бетонирование фундаментов
```

### Жизненный цикл опроса

1. **Старт**: бот читает ЕЖО-шаблон, находит активные коды с остатками > 0
2. **Опрос**: бот отправляет сообщение с запросом данных по категориям
3. **Сбор**: прорабы отвечают в группу, бот парсит:
   - VOR-коды (авто-детект `код = значение`)
   - Персонал/техника через QA-парсер
   - Материалы/планы через QA-парсер
4. **Сводка**: после каждого ответа — обновлённый статус опроса
5. **Закрытие**: авто-заполнение отсутствующих категорий → `fill_ejo.py` → отправка ЕЖО

### DB таблицы

**bot_poll_state:**
- id, chat_id, poll_date, status (active/closed), started_at, closed_at
- UNIQUE(chat_id, poll_date)

**bot_poll_residuals:**
- id, poll_id (FK → bot_poll_state), code, building, name, unit
- plan_volume, residual_volume, actual_today
- updated_by, updated_at
- UNIQUE(poll_id, code)

## ЕЖО (Ежедневный отчёт)

### Цикл
1. **Авто-заполнение**: fill_ejo.py собирает факты из БД + погода + персонал
2. **Ручная правка**: пользователь присылает исправленный .xlsx
3. **Diff по кодам**: `_update_template_from_correction()` сравнивает по VOR-кодам
4. **Шаблон**: обновляется из corrected версии
5. **Накопительные**: `yesterday_cum()` из предыдущего v1-файла

### Ключевые колонки в ЕЖО
| Колонка | Название | Назначение |
|---------|----------|------------|
| 11 | Кол-во | Плановый объём |
| 12-13 | Объём за сутки | План/факт за день |
| 15 | Накопительный за месяц | Месячный накопленный |
| 18 | Накопительный с начала СМР | **Итоговый факт** |
| 21 | Остаток | Оставшийся объём |

## Память (PostgreSQL)

- `bot_memory_messages` — все сообщения и документы
- `bot_memory_facts` — извлечённые факты (персонал, техника, инциденты, объёмы)
- `bot_poll_state` + `bot_poll_residuals` — состояние опросов
- `bot_schedule_phases` — график производства (8 этапов + 45 подзадач)

## Принципы

1. **Надёжность > Функциональность** — каждое изменение проверяется
2. **Минимальные изменения** — не трогать рабочие модули без необходимости
3. **Коммитить после каждого логического блока**
4. **Тестировать в песочнице** перед деплоем
5. **Не отправлять в боевую группу** без approval
