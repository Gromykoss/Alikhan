## 2. Хронология — документировать всё

- **CHRONOLOGY.md** — каждая значимая операция: что делали, как, какие баги исправили.
- **После каждого значимого изменения** — новая запись с датой и контекстом.

## 3. Тестирование — на песочнице перед боевой группой

- Все изменения сначала проверяются в песочнице `120363179621030401@g.us`
- Только после подтверждения — деплой на production `120363400682390076@g.us`
- Тестовые сценарии: опрос, ЕЖО, фото, QA-факты

## 4. Бэкап перед каждым деплоем — MANDATORY

**Формат:** читаемая дата `ММДД_ЧЧММ` (МесяцДень_ЧасМинуты). Никаких Unix timestamp.

```bash
# Перед правками — бэкап всех изменяемых файлов
cp router.py router.py.bak.$(date +%m%d_%H%M)
cp poll.py poll.py.bak.$(date +%m%d_%H%M)
cp main_waha.py main_waha.py.bak.$(date +%m%d_%H%M)
cp fill_ejo.py fill_ejo.py.bak.$(date +%m%d_%H%M)
cp templates/ЕЖО_шаблон.xlsx templates/ЕЖО_шаблон.xlsx.bak.$(date +%m%d_%H%M)
```

**Правило:** каждый деплой = сначала бэкап, потом правки, потом перезапуск бота.

## 5. Сравнивать логи до и после

После перезапуска бота — проверить `/tmp/alikhan.log`:
- Нет ли `[LOOP ERR]`
- Нет ли новых ошибок
- Бот видит группу (`Watching: ...`)

## 6. Инфраструктуру верифицировать при старте сессии

- Бот запущен? (`ps aux | grep main_waha`)
- Evolution API жив? (`docker ps | grep evolution`)
- PostgreSQL доступен? (проверить подключение к БД)
- Шаблон ЕЖО на месте? (`ls -la templates/ЕЖО_шаблон.xlsx`)

## 7. Pre-deploy чеклист

1. Бэкап изменяемых файлов (правило 4)
2. `python3 -m py_compile` — проверка синтаксиса
3. Перезапуск бота: `pkill -f main_waha; python3 main_waha.py`
4. Проверка логов: нет `LOOP ERR`, бот видит группу
5. Тест в песочнице: опрос → ответ → ЕЖО

## 8. Правило отката

Если правки сломали работающее:
```bash
cp *.bak.<дата> *.py
pkill -f main_waha; python3 main_waha.py
```

## 9. Баги → фиксация и исправление

Каждый баг фиксируется:
- Симптом: что именно не работает
- Причина: где в коде ошибка
- Исправление: что изменено
- Проверка: как убедились что работает

## 10. Никогда не говорить «сделано» не проверив

- Проверить лог бота после изменений
- Проверить что бот отвечает на команды
- Проверить что данные сохраняются в БД
- Проверить что ЕЖО генерируется корректно

## Архитектура

### Компоненты

| Компонент | Файл | Назначение |
|-----------|------|------------|
| **main_waha.py** | 549 строк | Главный цикл: приём сообщений, обработка команд, шаблоны |
| **router.py** | 89 строк | Маршрутизация: QA, Grok, DB lookup, проверка имени |
| **poll.py** | 574 строки | Управление опросом: запуск, парсинг ответов, закрытие |

| **db.py** | 610 строк | PostgreSQL: сообщения, факты, календарь, график |
| **handlers.py** | 567 строк | Обработчики: Grok-запросы, верификация ответов |
| **qa.py** | — | Парсинг QA-фактов (персонал, техника, материалы, инциденты) |
| **daily_snapshot.py** | — | Ежедневный снимок состояния стройки |

### Поток сообщения

```
Сообщение → main_waha.py
  → QA-парсер (факты: персонал, техника, инциденты)
  → Проверка на коды работ (2.4.2 — 104.3)
  → Проверка имени «алихан»
  → Router (Grok / DB lookup / расписание)
  → Ответ (текст или голос)
  
ВСЕ сообщения → save_message() в БД (пассивный режим)
```

### Триггеры команд

| Команда | Действие |
|---------|----------|
| `Алихан опрос` / `запускай опрос` | Запуск опроса остатков |
| `Алихан формируй ЕЖО` | Формирование ЕЖО (требует poll) |
| `Алихан формируй ЕЖО принудительно` | ЕЖО без проверок |
| `Алихан закончить опрос` | Закрытие опроса → ЕЖО |
| `2.4.2 — 104.3` | Обновление остатка (без «алихан») |
| `Алихан статус опроса` | Сводка собранных данных |

### Шаблон ЕЖО

- Путь: `templates/ЕЖО_шаблон.xlsx`
- Лист: «Ежедневный отчет» (столбцы A-W, строки 1-851)
- Ключевые столбцы: A(здание), C(код), D(наименование), L-N(суточные), O-Q(месячные), R-U(общие)
- **Важно:** `fill_ejo.py` НЕ очищает строки без работ за сегодня (сохраняет ручные правки)

### База данных

- PostgreSQL (`evolution_db`), Docker-контейнер `evolution-postgres`
- Таблицы: `bot_messages`, `bot_memory_facts`, `bot_poll_state`, `bot_poll_residuals`, `bot_calendar`, `bot_schedule`
