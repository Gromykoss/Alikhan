### Поток сообщения

```
Сообщение → main_waha.py
  → QA-парсер (факты: персонал, техника, инциденты)
  → Проверка на коды работ (2.4.2 — 104.3)
  → Проверка имени «алихан»
  → Router (Grok / DB lookup / расписание)
  → Ответ (текст или голос)
  
ВСЕ сообщения → save_message() в БД (пассивный режим)
```

## 11. Verify before claim — MANDATORY

**Никогда не утверждать факт без инструментальной проверки.**

- «Файл не существует» → `ls -la`
- Если не проверил — не говори. Молчание лучше лжи.

## 12. Правильный scope — сначала песочница

**Всегда начинать проверку с песочницы `120363179621030401@g.us`.**
- Production `120363400682390076@g.us` — только если песочница подтверждена.

## 13. Кумулятив — шаблон источник истины

**Накопленные значения (P, Q, S, T, U) — из шаблона, не вычисляются заново.**
- Не очищать кумулятивные колонки (16, 19, 21) перед чтением.
- Не добавлять v к prev_p/prev_s — шаблон уже включает сегодня (если был corrected EJO).
- `yesterday_cum()` — только если есть чистый вчерашний v1-файл.
- Исправленный ЕЖО от пользователя (`/tmp/corrected_ЕЖО_*.xlsx`) — эталон для сверки.

## 14. Regex — не только пробелы

**`\s*` матчит только whitespace. Между префиксом и кодом могут быть слова.**
- Использовать `[\w\s]*?` (lazy) вместо `\s*`.
- `is_plan` — проверять и category, и текст факта.
- Планы и работы могут сосуществовать для одного кода → не блокировать `cd not in dn`.

## 15. Фильтр категорий — защита от галлюцинаций Grok

**Grok возвращает VOR-коды в чужих категориях (монтаж, земляные работы).**
- `volumes()` — фильтровать `category IN ('объём', 'план')`.
- Остальные категории — игнорировать при расчёте объёмов.

## 16. Динамические таблицы — без хардкода

**Никаких `range(2)`. Сколько строк данных — столько и выводить.**
- Планы на завтра: `for i in range(len(items))`.
- Позиции стартовых строк вычислять динамически.

## 17. Age gate — сохранять ДО фильтра

**Сообщения должны попадать в БД независимо от возраста.**
- `save_message()` вызывать ДО `if now_ts - msg_ts > 300`.
- Старые сообщения — сохранять, но не обрабатывать.

## 18. Сверка с эталоном

**После генерации ЕЖО — сравнить с `corrected_ЕЖО_*.xlsx`.**
- Все значения P, Q, U должны совпадать с точностью до округления.
- Если corrected нет — сообщить пользователю.
