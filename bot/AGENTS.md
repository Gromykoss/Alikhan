### Методика заполнения ЕЖО (v3 — июль 2026)

**План на месяц (начало месяца):**
1. Команда `Алихан раскрой отчет` — бот присылает шаблон со ВСЕМИ раскрытыми строками
2. Пользователь заполняет столбец **O** (Накопительный объем за месяц → План)
3. Отправляет заполненный шаблон обратно в песочницу
4. Бот читает столбец O — только строки с планом > 0 попадают в ежедневный опрос

**Ежедневный опрос:**
1. `Алихан запускай опрос` — бот формирует список работ по столбцу O (месячный план > 0)
2. Прорабы отвечают: `код = объём` (например: `3.1.1 = 50`)
3. `Алихан статус опроса` — сводка собранного
4. `Алихан закончить опрос` — закрытие → формирование ЕЖО

**Принцип:** Столбец O — единственный источник истины для фильтра опроса.
Нет O > 0 → нет в опросе. Никаких догадок по графику.

### Триггеры команд

| Команда | Действие |
|---------|----------|
| `Алихан раскрой отчет` | Раскрыть все строки шаблона → прислать файл (для месячного плана) |
| `Алихан опрос` / `запускай опрос` | Запуск опроса по столбцу O (план > 0) |
| `Алихан формируй ЕЖО` | Формирование ЕЖО |
| `Алихан формируй ЕЖО принудительно` | ЕЖО без проверок |
| `Алихан закончить опрос` | Закрытие опроса → ЕЖО |
| `код = объём` | Обновление остатка (парсится авто) |
| `Алихан статус опроса` | Сводка собранных данных |
| `Алихан календарь` | Ближайшие напоминания |
| `Алихан напомни ...` | Создать напоминание |

## 14. Regex — не только пробелы

**`\s*` матчит только whitespace. Между префиксом и кодом могут быть слова.**
- Использовать `[\w\s]*?` (lazy) вместо `\s*`.

## 15. Фильтр категорий — защита от галлюцинаций Grok

**Grok возвращает VOR-коды в чужих категориях (монтаж, земляные работы).**
- `volumes()` — фильтровать `category IN ('объём', 'план')`.
- Остальные категории — игнорировать при расчёте объёмов.

## 16. Динамические таблицы — без хардкода

**Никаких `range(2)`. Сколько строк данных — столько и выводить.**
- Планы на завтра: `for i in range(len(items))`.
- Позиции стартовых строк вычислять динамически.

## 17. Age gate — сохранять ДО фильтра

**Сообщения должны попадать в БД независимо от возраста.**
- `save_message()` вызывать ДО `if now_ts - msg_ts > 300`.
- Старые сообщения — сохранять, но не обрабатывать.
