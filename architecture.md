## Поток сообщения

### Входящее сообщение

1. WhatsApp → evolution-api (poller 3s)
2. main_waha.py демон — Evolution API `/chat/findMessages` (poll 3s)
3. **Guard:** skip fromMe, skip age > 5min
4. **Медиа-обработка:**
   - Фото → сохранение в БД (`building`, `msg_id`)
   - Документы → base64 → `localhost:8099` экстрактор → если ЕЖО .xlsx → обновление шаблона
   - Голосовые → STT транскрибация → сохранение текста в БД
5. **QA Parser:** если есть данные (персонал, техника, VOR-коды) → save to `bot_memory_facts`
6. **Poll handlers (до Grok):**
   - `начать опрос` → `start_poll()` → опрос с остатками работ
   - `закончить опрос` → `close_poll()` → ЕЖО
   - `статус опроса` → `get_poll_status()` → сводка
   - VOR-коды в тексте → `parse_poll_reply()` → update residuals
   - `заполни ежо` → проверка опроса + генерация ЕЖО
7. **Name check:** если нет «алихан» → IGNORE
8. **Command detection:** если CMD → skip Grok
9. **Router:** Schedule → Weather → DB → Grok
10. **Verification:** `verify.py` (REJECT <40 / FLAG <70 / OK 90+)
11. **Reply:** `send_msg()` / `send_voice()` / `sendMedia` (document, audio)

### Напоминание календаря (НЕ РАБОТАЕТ — не портирован из n8n)

1. **Schedule Trigger:** каждую минуту
2. **Postgres SELECT:** `bot_calendar_events` WHERE `remind_at <= NOW()` AND `reminder_sent = FALSE`
3. **Format:** сборка текста напоминания
4. **HTTP Request:** отправка в WhatsApp
5. **Postgres UPDATE:** `reminder_sent = TRUE`

## Команды пользователя

| Команда | Действие |
|---------|----------|
| «Алихан начать опрос» | Запустить опрос, показать остатки работ |
| «Алихан закончить опрос» | Закрыть опрос → авто-заполнение → ЕЖО |
| «Алихан статус опроса» | Сводка собранного/недостающего |
| «Алихан заполни ежо» | Проверить опрос → сформировать ЕЖО |
| «Алихан напомни _текст_ _дата_ _время_» | Создать событие календаря (пример: «Алихан напомни совещание 15.07 14:00») |
| «Алихан календарь» | Показать события на неделю |
| Ответ с VOR-кодом (`2.1.1 = 85.5`) | Авто-обновление остатков в опросе |
| Любое сообщение с данными | QA-парсер: персонал, техника, инциденты |
| Голосовое сообщение | STT → текст → обработка |
| Фото с подписью (АБК, Общежитие…) | Сохранение в БД с тегом здания |
| Файл .xlsx (ЕЖО) | Diff → обновление шаблона |
| Любой вопрос | Grok (xAI) → verify → ответ |
