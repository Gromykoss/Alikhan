## 08.07.2026 (продолжение) — Fix: фото из боевой группы + очистка суточных колонок

### Fix 1: Слушатель боевой группы не захватывал сообщения
**Причина:** окно 10 минут + нет персистентности seen_ids. Все сообщения старше 10 мин пропускались.

**Исправление:**
- `prod_seen_ids.json` — персистентное хранение обработанных ID (как seen_ids.json для песочницы)
- Первый запуск: окно 24 часа, лимит 20 сообщений (seed)
- Последующие: без окна, лимит 5, только новые
- Результат: 5 фото захвачено сразу после перезапуска

### Fix 2: Шаблонное загрязнение суточных колонок (3.1.4 = 798)
**Причина:** `_update_template_from_correction` копирует исправленный ЕЖО как шаблон. Суточные данные из правки «застывали» в шаблоне и показывались как сегодняшняя работа.

**Исправление:** в `fill_ejo.py` — очистка суточных колонок (L=12, M=13, N=14) для ВСЕХ строк с VOR-кодами, не только для строк с работой сегодня. Накопительные колонки (P, Q, S, T, U) сохраняются.

### Файлы

### Анализ логики ЕЖО (зафиксировано)
**Источники данных:**
- Суточный объём: QA-факты из `bot_memory_facts` (через Grok)
- Накоп за месяц: вчерашний Excel (колонка P) + сегодня
- Накоп СМР: вчерашний Excel (колонка S) + сегодня
- План: шаблон (колонка K)
- Guard: очищаются только строки с работой сегодня, остальные — не трогаются

**Колонки:**
- L, M (12,13) — факт за сутки
- N (14) — 1
- P (16) — накоп за месяц
- Q (17) — % за месяц = (prev_p+v)/mp
- S (19) — накоп СМР
- T (20) — % СМР = (prev_s+v)/tp
- U (21) — остаток = k_plan − prev_s − v

## 06.07.2026 — T-128: Восстановлен функционал опроса (poll) остатков работ

### Что сделано
- **Создан модуль `bot/poll.py`**: управление опросом, сбор остатков работ по VOR-кодам
- **DB: новые таблицы** `bot_poll_state` + `bot_poll_residuals` для отслеживания опросов
- **main_waha.py**: интеграция poll.py вместо старого вызова `smart_evening_check.py`
  - `Алихан начать опрос` → создаёт poll в БД, проверяет собранные QA-факты, формирует запрос с остатками работ по активным VOR-кодам
  - Авто-детект ответов прораба: парсинг VOR-кодов (`код = объём`), обновление `bot_poll_residuals`
  - `Алихан статус опроса` / `что собрано` → сводка собранного и недостающего
  - `Алихан закончить опрос` → авто-заполнение недостающих категорий → генерация ЕЖО
  - `Алихан заполни ежо` → проверка активного опроса, если данные не все → подсказка
- **router.py**: расширен `cmd_words` (статус опроса, опрос стоп, и т.д.)
- **Убран прямой вызов** `smart_evening_check.py` — вся логика в poll.py

### Архитектура опроса
1. `start_poll()` — читает активные остатки из шаблона/последнего ЕЖО, сохраняет в `bot_poll_residuals`
2. `build_poll_message()` — формирует сообщение: что собрано (✅), что нет (❌), какие остатки нужны
3. `parse_poll_reply()` — парсит ответ прораба: VOR-коды, персонал, технику, планы
4. `get_poll_status()` — статус опроса, сводка собранного
5. `close_poll()` — авто-заполнение → генерация ЕЖО через `fill_ejo.py`
